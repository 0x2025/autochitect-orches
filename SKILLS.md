# Skills Documentation

## Node Backend Skills
- Careful planning before work
- Evaluate feasibility via self-challenges and critical thinking
- Ensure compatibility and don't slow down developer experience
- Write tests before code (TDD approach)
- Recognize SUT (System Under Test) with proper naming conventions
- Iterate implementation until tests pass and code is simple/modular

## Node Performance Skills
- Review functions, aggregations, database connections for bottlenecks
- Ensure async usage and query limits
- Avoid raw input in queries
- Manage deserialization/serialization of large objects

## Go Backend Skills
- Clean code, high performance, testable code, modular design
- TDD approach for testability
- Abstract external systems for testing

These skills document the expected behaviors for AI agents working on our agent-first repository. They should guide the development of coding agents to ensure they make the right decisions and produce high-quality code.
